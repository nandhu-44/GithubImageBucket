Github Image Bucket API

This API stores project files in a GitHub repository under uploads/<projectName>/. It supports multi-project upload, file listing, and file deletion with API key protection on all backend routes.

Base URL: http://localhost:5000 Auth: x-api-key Default Branch: main

Authentication

All backend routes use API key middleware.

  • Header: x-api-key: <API_KEY>
  • Query fallback: ?x-api-key=<API_KEY>
ConditionStatusResponse
Missing API key401{"error":"x-api-key is required"}
Invalid API key403{"error":"Invalid API key"}

Project Naming

projectName is normalized to lowercase and validated by:

^[a-z0-9][a-z0-9-_]{1,62}$
  • 2 to 63 characters
  • Allowed: lowercase letters, numbers, -, _
  • Must start with lowercase letter or number

Backend Endpoints

GET/health

Health check endpoint.

{ "status": "ok" }
POST/projects

Create project placeholder in GitHub.

{ "projectName": "demo-app" }

201 created, 200 already exists, 400 invalid projectName

{
  "projectName": "demo-app",
  "created": true,
  "placeholder": "https://raw.githubusercontent.com/<owner/repo>/<branch>/uploads/demo-app/_placeholder.txt"
}
GET/projects

List project directories from uploads/.

{ "projects": ["demo-app", "marketing-assets"] }
POST/projects/:projectName/upload

Upload file using multipart/form-data key file.

{
  "publicUrl": "https://raw.githubusercontent.com/<owner/repo>/<branch>/uploads/demo-app/logo.png",
  "projectName": "demo-app",
  "fileName": "logo.png"
}
CaseStatusResponse
Invalid project name400{"error":"Invalid project name"}
Missing file400{"error":"File is required under key 'file'"}
Upload failure500{"error":"Failed to upload the file"}
GET/projects/:projectName/files

List files for a project (excluding _placeholder.txt).

[
  {
    "name": "logo.png",
    "url": "https://raw.githubusercontent.com/<owner/repo>/<branch>/uploads/demo-app/logo.png"
  }
]
CaseStatusResponse
Invalid project name400{"error":"Invalid project name"}
Project not found404{"error":"Project not found"}
List failure500{"error":"Failed to retrieve files"}
DELETE/projects/:projectName/files

Delete one file from a project folder.

{ "fileName": "logo.png" }
{ "message": "File \"logo.png\" deleted successfully" }
CaseStatusResponse
Invalid project name400{"error":"Invalid project name"}
Missing fileName400{"error":"fileName is required"}
Delete placeholder file400{"error":"Cannot delete placeholder file"}
File/project not found404{"error":"File or project not found"}
Delete failure500{"error":"Failed to delete the file"}

Frontend Proxy

Frontend can call Next.js proxy route at frontend/src/app/api/[...path]/route.js:

/api/health
/api/projects
/api/projects/:projectName/upload
/api/projects/:projectName/files
  • Injects backend API key from BACKEND_API_KEY
  • Forwards multipart uploads with formData()
  • Forwards JSON/binary payloads and upstream status
Proxy ErrorStatusResponse
Missing BACKEND_API_KEY500{"error":"Missing BACKEND_API_KEY in frontend environment"}
Backend unreachable502{"error":"Failed to reach backend service"}

Environment Variables

VariableScopeRequiredDescription
PORTBackendNoExpress server port (default 3000)
API_KEYBackendYesRequired API key for backend middleware
GITHUB_TOKENBackendYesGitHub token with contents read/write access
GITHUB_REPOBackendYesRepository in owner/repo format
GITHUB_BRANCHBackendNoTarget branch (default main)
BACKEND_BASE_URLFrontendNoProxy target URL (default http://localhost:5000)
BACKEND_API_KEYFrontendYesAPI key injected by proxy

Examples

cURL

curl -H "x-api-key: YOUR_API_KEY" http://localhost:5000/health

curl -X POST http://localhost:5000/projects \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"projectName":"demo-app"}'

curl -X POST http://localhost:5000/projects/demo-app/upload \
  -H "x-api-key: YOUR_API_KEY" \
  -F "file=@./logo.png"

curl -H "x-api-key: YOUR_API_KEY" http://localhost:5000/projects/demo-app/files

curl -X DELETE http://localhost:5000/projects/demo-app/files \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"fileName":"logo.png"}'

PowerShell

$base = "http://localhost:5000"
$headers = @{ "x-api-key" = "YOUR_API_KEY" }
$headersJson = @{ "x-api-key" = "YOUR_API_KEY"; "Content-Type" = "application/json" }

Invoke-RestMethod -Method Get -Uri "$base/health" -Headers $headers

$createBody = @{ projectName = "demo-app" } | ConvertTo-Json
Invoke-RestMethod -Method Post -Uri "$base/projects" -Headers $headersJson -Body $createBody

$deleteBody = @{ fileName = "logo.png" } | ConvertTo-Json
Invoke-RestMethod -Method Delete -Uri "$base/projects/demo-app/files" -Headers $headersJson -Body $deleteBody

Operational Notes

  • Backend startup fails immediately if GITHUB_TOKEN, GITHUB_REPO, or API_KEY is missing.
  • Project folders are represented by _placeholder.txt.
  • Uploading an existing filename performs SHA-aware update.
  • CORS and JSON body parsing are enabled globally in Express.
